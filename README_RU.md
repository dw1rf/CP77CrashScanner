# CP77 Crash Scanner

Десктопный инструмент для диагностики вылетов и конфликтов модов в **Cyberpunk 2077**.  
Сканирует логи **Mod Organizer 2** и **Vortex**, классифицирует ошибки, проверяет версии фреймворков и формирует отчёт.

> **[Скачать последний релиз →](https://github.com/dw1rf/CP77CrashScanner/releases/latest)**  
> Также доступно на **[Nexus Mods](https://www.nexusmods.com/cyberpunk2077/mods/30924)**

---

## Возможности

- **Авто-сканирование** при запуске, если инстанс MO2 уже настроен
- **Определение причин вылетов** — показывает конкретные моды-виновники
- **Детектор конфликтов в r6** *(новое в 1.2)* — разбирает логи `redscript`, `TweakXL` и `ArchiveXL` и называет моды, которые перезаписывают чужие методы/записи или ссылаются на отсутствующий код
- **Проверка версий фреймворков** — RED4ext, CET, ArchiveXL, TweakXL, Codeware, redscript
- **Вкладка «Не загрузилось»** — готовый список несовместимых модов для публикации
- **Поддержка Vortex** — определяется автоматически, настройка не нужна
- **Двуязычный интерфейс** — Русский / English, переключается в любой момент
- **Очистка логов** *(новое в 1.3)* — удаление всех просканированных лог-файлов и крэш-дампов в один клик, с подтверждением «Да/Нет». Убери старые логи, запусти игру и воспроизведи вылет, затем сканируй чистый набор
- **Экспорт в один файл** — полный отчёт с сырыми логами в одном `.txt`
- **Копирование текста** — все панели с результатами выделяемые и копируемые

## Требования

- Windows 10 / 11
- Cyberpunk 2077 (рекомендуется с RED4ext)
- Python не нужен — один портативный `.exe`

## Использование

1. Скачай `CP77CrashScanner.exe` из [Releases](https://github.com/dw1rf/CP77CrashScanner/releases)
2. Запусти — авто-сканирует, если путь к MO2-инстансу уже сохранён
3. Укажи папку **MO2-инстанса** и/или папку **игры**
4. Нажми **СКАНИРОВАТЬ**
5. Изучи вкладки; нажми **ОТЧЁТ В 1 ФАЙЛ** для экспорта

## Вкладки

| Вкладка | Что показывает |
|---------|----------------|
| Сводка | Количество ошибок, топ модов, причины вылетов с именами модов |
| По модам | Все моды с раскрываемым списком ошибок |
| Не загрузилось ⚠ | Моды, которые не загрузились или не скомпилировались |
| Конфликты r6 ⚔ | Моды, перезаписывающие один метод/запись + ошибки компиляции redscript |
| Все ошибки | Все уникальные ошибки и предупреждения без дублей |
| Совместимость | Версии фреймворков, статус Vortex, просканированные пути |
| Дампы и логи | Крэш-дампы и лог-файлы с временными метками |

## Определяемые причины вылетов

| Причина | Описание |
|---------|----------|
| `ошибка компиляции redscript` | Скрипт-мод не компилируется — чёрный экран при загрузке |
| `плагин RED4ext не загрузился` | DLL-мод несовместим с текущим патчем |
| `несовместимая версия` | Мод собран под другой патч |
| `отсутствует класс/метод` | Отсутствует зависимость-фреймворк |
| `краш движка (access violation)` | Конфликт архив-модов или повреждённый `.archive` |
| `отсутствует .archive` | Неполная установка мода |
| `NCA: сбой стейт-машины` | Вылеты Night City Allies в конкретных локациях |

## Детектор конфликтов в r6 *(новое в 1.2)*

Штатные детекторы конфликтов в MO2/Vortex видят только перезапись файлов в `archive/pc/mod`. Конфликты внутри `r6` они не замечают — те всплывают только при компиляции `redscript`. Сканер читает эти логи и называет виновных модов на вкладке **Конфликты r6 ⚔**:

| Источник | Что ловит |
|----------|-----------|
| `redscript` | Два мода делают `@replaceMethod` на один метод (активен только один) — показывает проигравший мод и целевой класс |
| `redscript` | Ошибки компиляции — отсутствующий класс/метод, unresolved-ссылки из устаревшего скрипт-мода |
| `TweakXL` | Конфликты/переопределения записей и проблемы зависимостей |
| `ArchiveXL` | Отсутствующие зависимости и конфликты ресурсов |

Пример вывода: `[CONFLICT] [redscript] quickhacks_sort_by_slot @replaceMethod(RPGManager): this method replacement overwrites a previous annotation…`

> Важно: redscript называет только *проигравший* мод в конфликте перезаписи (чью замену отбросили) — «победителя» движок в лог не пишет. Но этого достаточно, чтобы найти виновника и отключить его либо поправить порядок загрузки.

## Конфигурация

Настройки сохраняются в `scanner_config.json` рядом с `.exe`:

```json
{
  "instance": "C:\\MO2\\MO2_Cyberpunk",
  "game": "C:\\Games\\Cyberpunk 2077",
  "lang": "ru",
  "recommended": {
    "RED4ext": "1.27.0",
    "Cyber Engine Tweaks": "1.35.0"
  }
}
```

## Сборка из исходников

Требования: Python 3.12+, Windows 10/11.

```bash
pip install -r requirements.txt
build_windows.bat
```

Результат: `dist\CP77CrashScanner\CP77CrashScanner.exe` и релизный ZIP. Сборка использует Nuitka standalone без PyInstaller, UPX, обфускации и самораспаковывающегося установщика. В архив включён `SHA256SUMS.txt` с проверяемыми контрольными суммами.
